STEP 1: CONFIGURE (30 seconds)

  Edit CONFIG.env and enter your website details:
  
  $ nano CONFIG.env
  
  Change these 3 lines:
  
  TARGET_WEBSITE_URL=http://your-actual-website.com
  PUBLIC_IP_OR_DOMAIN=your-domain.com
  PUBLIC_PORT=8080

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 2: DEPLOY (1 minute)

  Press one button to deploy:
  
  $ bash start_waf.sh
  
  The script automatically:
  ✓ Builds Docker images
  ✓ Starts all services
  ✓ Enables auto-restart on failure
  ✓ Launches monitoring daemon
  ✓ Logs everything

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 3: VERIFY (30 seconds)

  Open your website in a browser:
  
  http://<your-ip or domain>:8080
  
  Your website should load normally.
  
  Test that attacks are blocked:
  
  curl "http://<your-ip>:8080/path?q=' OR 1=1"
  
  Expected response: 403 Forbidden ✓

  View production logs:
  $ tail -f waf_production.log
  
  Check services are running:
  $ docker-compose ps

  $ tail -f nginx/logs/access.log
  
  View only blocked attacks:
  $ tail -f nginx/logs/access.log | grep " 403 "
